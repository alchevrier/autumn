# Autumn Sandbox

This module serves as the **integration test environment** for the `autumn-compiler-plugin`.

## Why does this exist?
Testing Kotlin Compiler Plugins (especially for K2 / Kotlin 2.1+) using mock compilation frameworks like `kotlin-compile-testing` is currently brittle due to internal reflection and scripting classpath issues. 

Instead of relying on a broken or unstable test suite, this `autumn-sandbox` module acts as a real-world consumer. It directly applies the compiler plugin through Gradle `-Xplugin` arguments. If the compiler plugin logic works, this module's compilation will enforce the rules natively.

## How to prove the compiler plugin works

1. Open `src/main/kotlin/dev/autumn/sandbox/AllocationCases.kt`.
2. This file contains a mix of allowed allocations (primitives, strings, exceptions, and objects inside `@LongLived` scopes) and explicitly forbidden heap allocations.
3. Uncomment any of the lines marked as `ERROR` (e.g., `val list = ArrayList<String>()` or `val state = AppState()`).
4. Run the compilation task from the root of the repository:

```bash
./gradlew :autumn-sandbox:compileKotlin
```

5. You should see the build **fail** with a hard compiler error generated by our K2 plugin, looking similar to this:

```text
e: Heap allocation detected in strict zero-allocation scope: kotlin.collections.ArrayList. Annotate with @LongLived if this is a safe long-lived allocation.

> Task :autumn-sandbox:compileKotlin FAILED
```

This proves that the zero-allocation architecture constraints defined in our ADRs are being actively and strictly enforced at compile time.
