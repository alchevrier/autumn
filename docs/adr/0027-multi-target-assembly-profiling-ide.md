# ADR 0027: Multi-Target Assembly Profiling and IDE Drill-Down

## Status
Accepted

## Context
Autumn's pipeline telemetry currently maps overall cycle budgets at the function level (`@Observe` and `@CycleBudget`). However, when building cross-platform (KMP) real-time systems, 40 cycles on a JVM implies vastly different operations than 40 cycles on an Apple M3 (ARM64) or an AMD Zen 4 (x86_64).

To provide absolute transparency, the IDE Performance Center needs to act not just as a topological graph, but as an integrated *Compiler Explorer (Godbolt)*. Developers must be able to click on any node in their dataflow pipeline and see exactly how the Kotlin Intermediate Representation (IR) translates into the target platform's lowest-level instructions, complete with line-by-line micro-architectural cycle costs.

## Decision
We will expand the `autumn-ide-plugin` and `TopologyComponent` telemetry schema to support **Multi-Target Low-Level Drill-Down**.

1. **Target-Specific Translation Layers**: The Performance Center UI graph will allow developers to drill down into any `@Observe` or `@BoundaryChannel` node and select a deployment target to view the generated instructions:
   - **JVM Target (Server)**: Displays standard JVM Bytecode (e.g., `IASTORE`, `INVOKESTATIC`, `VarHandle.setRelease()`).
   - **Native (Linux/x86_64)**: Displays LLVM Bitcode and generated `x86_64` assembly.
   - **Native (iOS/Android ARM64)**: Displays translated `ARM64` assembly natively.
   
2. **Instruction-Level Cycle Mapping**: The telemetry generated by the K2 compiler's `CycleBudgetVisitor` will be enhanced to map cycle expenditures directly to individual opcodes/assembly lines within the drill-down view.

3. **Domain-Specific Visualization**: The IDE will visually segregate these pipelines. `@InjectTopology` dictates the root of an execution core map. Each root gets its own isolated tab in the UI, rendering its physical boundary-to-boundary pipeline.

## Consequences
- **Positive:** Unprecedented transparency. Developers no longer need to use external disassemblers (`javap`, `objdump`) or guess how Kotlin handles their loops on an M3 vs a Xeon.
- **Positive:** Seamlessly proves Autumn's zero-allocation assertions directly in the IDE natively across platforms.
- **Negative:** Telemetry JSON files (`topology.json`) footprint will increase as we embed operation-level metadata for multiple architectural targets.
