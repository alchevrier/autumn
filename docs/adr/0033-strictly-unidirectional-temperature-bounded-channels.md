# 33. Strictly Unidirectional and Temperature-Bounded Channels

Date: 2026-07-02

## Status
Accepted

## Context
As Autumn scales natively out to external file descriptors for network bridging and process boundaries (e.g. `AF_XDP`, POSIX UDP, `/dev/shm` IPC Mmaps), modeling assumptions in the declarative Kotlin API and the K2 Compiler IR pipeline have become ambiguous. 

Previously, `AutumnChannel` could technically be conceptualized as bidirectional—an abstraction representing an execution boundary where information is ferried around. An `@IpcGateway`-annotated channel, for instance, might be declared as "local pipeline interacting with external shared memory." 

However, at the microsecond/nanosecond levels of hardware cache lines and cycle budgeting, bidirectional or abstract channels are a massive anti-pattern:
1. **Ambiguous Geometry**: When the K2 compiler extracts the mathematical footprint of a strictly bounded `value class` inline struct (e.g., `OrderEvent(index: Int)`), it assumes a contiguous uniform struct pool array injected uniformly over `AutumnMemoryBank`. If the pipeline writes data out of local RAM into a POSIX `/dev/shm` boundary natively mapped, an internal structure identifier loses its context (e.g. is this index local space or OS space?). 
2. **Backpressure Starvation**: If a hot computational kernel uses a bidirectional socket, any unread backpressure generated by external elements (unbounded networking layer limits, physical Ethernet congestion, OS thread jitter from detached Reader processes) immediately imposes Wait-states upon the pristine `HardwareOscillator` logic. Wait-states nullify L1 pinning guarantees. 
3. **Loss of Compilation Reasonability**: If structural boundaries overlap, we cannot guarantee DAG (Directed Acyclic Graph) determinism, rendering the `CycleBudgetVisitor` and WCET extraction impossible. 

## Decision
We establish exactly how hardware and processes flow across physical mediums in software via the concept of **Temperature Directionality**:

### 1. Ingress = Hot (`@BoundaryChannel`)
When data flows *from* an external boundary (Networking, Shared Memory ingress) *into* the computational hot path, it is natively **Hot**. 
- The `HardwareOscillator` L1 Scheduler directly binds an active clock tick to aggressively poll this buffer without operating system intervention (`sched_yield()` equivalent spins on the cache line pointer).
- It consumes WCET budget. 
- Struct allocations originate here and geometrically govern the monolithic offset sizes. 

### 2. Egress = Cold (`@ColdChannel`)
When data completes FSM transition inside the algorithmic Hot Path and must leave the physical system (Outbound Networking, Log flushing, persistence files, Egress IPC to an untrusted telemetry/compliance container process), it is natively **Cold**. 
- The primary computational Hot Path performs an explicitly allocated **Copy** onto this channel. 
- It pushes the frame as purely Fire-And-Forget FSM multicasts. The computation immediately resumes executing the next hot loop independently.
- The `HardwareOscillator` completely ignores evaluating this bound. Instead, asynchronous physical drivers explicitly mapped as background physical threads consume these offsets sequentially and drain to file descriptors blocks without backpressuring the Hot CPU core. 

### 3. Structural Pointer Segregation
Because of unidirectional mandates, the K2 IR Transformer natively strips all structures bound strictly behind `@IpcGateway` constraints exclusively from the internal `AutumnMemoryBank` global topology computation. They are independently segregated dynamically to external mapped boundaries with a clean `CPointer` overlay without injecting arbitrary `index + block_id` pointer dereferencing conditional logic directly into fast computational cycles.

## Consequences
- **Positive**: Complete DAG guarantees globally extending beyond process-level into whole-cluster network dataflow modeling.
- **Positive**: Safe IPC memory segmentation scaling natively via Kubernetes `/dev/shm` emptyDirs with mathematically impossible performance variance on local hardware contexts. 
- **Negative**: Developers cannot merely pass `Int` indices out asynchronously to the logger. They must define unique payload-specific boundaries and strictly allocate copied translations prior to releasing frames bounding strictly cold executions.
